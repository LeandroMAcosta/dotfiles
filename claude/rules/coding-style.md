## Coding Style

- Write all code in English: variables, functions, classes, comments.
- User-facing text in Spanish (or bilingual where i18n applies).
- Prefer immutability: `const` over `let`, frozen objects where practical.
- Keep files under 300 lines. Split when logic is distinct enough.
- No `console.log` in committed code — use proper logging or remove before commit.
- No emojis in code or comments.
